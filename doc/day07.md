### RESOLUCIÓN
#### Parte1
Esta primera parte se lleva a cabo con el método executeA(), el cual llama a solve(), cuyo funcionamiento consiste en un Stream que pasa por todas las filas de la entrada. En cada una de ellas, comprueba si alguna de las bolas del ballArray() choca con un ^ en la siguiente fila; si eso ocurre, actualiza el ballArray para la siguiente iteración y, además, devuelve true, lo que hace que el Stream cuente 1 (count()).
#### Parte2
En la segunda parte utilizaremos el método executeB(), el cual, además de la lógica necesaria para crear el ballArray y modificarlo (al igual que en la parte 1), se apoya en los siguientes métodos:
* generateArray(): Se encarga de transformar el array inicial de enteros a un array de tipo long[], a partir del cual generaremos un LongStream para sumar todas las combinaciones finales.
* followTimelines(): Su función es procesar la evolución de las rutas para devolver el array final. Para ello, sigue un método similar al de la parte 1: se ejecuta un Stream sobre las filas de la entrada y se actualiza el estado en una variable temporal (rowTemp). Esta variable acumula las trayectorias y se vuelca mediante una copia de memoria sobre el ballArray para ser utilizada en la siguiente iteración.
